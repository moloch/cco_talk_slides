Collective Code Ownership

It's a rule of eXtreme Programming

Origins

Wikiwiki (Ward Cunningham)

Everyone knows
the codebase

and is responsible for its quality!

The benefits

Knowledge sharing

Everyone will know most of the code

Responsability

Encourages each developer to feel responsible for the quality of the whole

Better working environment

Developers talk with each other

No single point of failure

Velocity and code quality are up to the team, not the single developer.

How to make it work

Build a team

A group of people is not a team

Automated tests

Share knowledge, avoid regression, enhance confidence.

Rotation +

Code reviews

Different eyes will find bugs

Regular meetings
